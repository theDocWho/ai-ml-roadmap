GroupBy: Split → Apply → Combine

The workhorse of data analysis: "for each category, what's the average / total / count?" Every groupby is the same three-step dance — and once you see the steps, SQL's GROUP BY and pandas click into the same shape.

split–apply–combineaggregation like SQL GROUP BY

The pattern

You have rows; you want a summary per category. Average score per city. Total spend per customer. Number of orders per day. The naive way is a loop with a dictionary of running totals. Pandas gives you one expression instead, built on the split–apply–combine idea (the same one behind SQL's GROUP BY and MapReduce):

  1. Split the rows into groups by the value of a key column (all the NYC rows together, all the LA rows together, …).
  2. Apply a function to each group independently — usually an aggregation that reduces the group to one number (mean, sum, count, max…).
  3. Combine the per-group results back into a single Series or DataFrame, indexed by the group key.

In code that's just: df.groupby('city')['score'].mean() — read it as "split by city, take the score column, apply mean to each group." Change the key, the column, or the function below and watch all three stages update:

Beyond a single number per group

Aggregation is the common case, but groupby does more:

  • Several aggregations at once: df.groupby('city')['score'].agg(['mean', 'max', 'count']) returns a DataFrame with a column per statistic.
  • Different functions per column: df.groupby('city').agg(avg_score=('score','mean'), n=('name','count')) — named aggregations, the tidy modern style.
  • Multiple keys: df.groupby(['city', 'plan']) splits on the combination, giving a hierarchical (Multi-)index.
  • Transform (not reduce): df.groupby('city')['score'].transform('mean') returns a result the same length as the input — each row gets its group's mean — perfect for "subtract the group average" feature engineering.

Why it matters

Grouped aggregation is most of exploratory data analysis and a huge chunk of feature engineering: per-user averages, per-category rates, per-day counts, rolling cohort stats. It's declarative (you say what, not how), it runs in fast vectorized C, and the split–apply–combine framing transfers directly to SQL and Spark. The main gotcha is forgetting that an aggregation moves the key into the index — if you'd rather keep it as a normal column, tack on .reset_index() (or pass as_index=False).

Takeaways: groupby = split rows by a key → apply a function per group → combine into a result indexed by the key. df.groupby(k)[c].f() is the core; reach for .agg(...) for several stats and .transform(...) when you need a same-length result.
