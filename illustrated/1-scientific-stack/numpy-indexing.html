Indexing: Slices, Masks & Fancy

Three ways to pull data out of a NumPy array — and the one distinction that bites everyone: slices return a view (a window onto the same memory), while masks and fancy indexing return a copy.

slicingboolean masks fancy indexingview vs copy

Why NumPy indexing is more than Python's

Python lists give you a[2] and a[1:5]. NumPy keeps those and adds two superpowers that make data selection loop-free: boolean masking ("give me every element where a condition holds") and fancy indexing ("give me exactly these positions, in this order"). Together with slicing they cover almost everything you'll do when filtering rows, picking columns, or selecting examples for a batch. Pick a mode below; the array stays fixed so you can compare how each one selects.

1 · Slicing — a[start:stop:step] returns a view

Slicing works like Python's (stop is exclusive; negative indices count from the end; a[::-1] reverses; omitting an endpoint means "to the edge"). The crucial NumPy difference: a slice is a view, not a copy. It's a lightweight window onto the same underlying memory, so it's free to make — but writing to it writes through to the original array:

a = np.arange(10)
part = a[2:5]   # a view
part[:] = 0    # also zeros a[2:5]!
a    # → [0 1 0 0 0 5 6 7 8 9]

This is a feature — it lets you modify a sub-block of a giant array cheaply — but it surprises people who expect a copy. When you genuinely need an independent copy, say so: a[2:5].copy().

2 · Boolean masks — select where a condition is True

Compare an array to something and you get a boolean array of the same shape: a > 5 → [False, True, …]. Index with that mask and NumPy returns every element where the mask is True, as a new 1-D array (a copy). This is how you filter: a[a > 5], or combine conditions with & / | (use parentheses — a[(a > 2) & (a < 8)]). You can also assign through a mask: a[a < 0] = 0 clamps negatives. 3 · Fancy indexing — pick exact positions, in any order

Pass a list (or array) of integer positions and NumPy returns those elements — in the order you asked, repeats and all: a[[4, 0, 4]] gives the 5th, 1st, then 5th element again. Negative indices work too. Fancy indexing always returns a copy, and its result takes the shape of the index array — which is how you'd, say, gather a mini-batch of rows X[[7, 2, 9]] or reorder data by a sorted-index array from np.argsort.

View vs copy — the rule to remember

This trips up even experienced users, so make it a reflex:

  • Basic slicing → view. Cheap, shares memory, writes propagate to the original.
  • Boolean masking → copy. A fresh array of the selected elements.
  • Fancy (integer-list) indexing → copy. A fresh array shaped like your index.

Unsure at runtime? arr.base is None tells you whether an array owns its data (a copy) or is a view onto someone else's. When in doubt and you don't want spooky action at a distance, add .copy().

Takeaways: slicing (a[i:j:k]) is a cheap view — edits write back; boolean masks (a[a>k]) filter by condition into a copy; fancy indexing (a[[i, j, k]]) gathers exact positions in your chosen order into a copy. Masks and fancy indexing are how you filter and gather data without ever writing a loop.
