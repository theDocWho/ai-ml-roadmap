Self-Attention, Step by Step

The one operation that powers every transformer: each token looks at every other token and pulls in what's relevant. It's just three matrices — Query, Key, Value — and the formula softmax(Q·Kᵀ / √d)·V.

The problem attention solves

To understand a word you need its context. In "the cat ate the fish," the meaning of "ate" is tied to its subject ("cat") and object ("fish"), which sit several positions away. Older models pushed information along the sequence step by step (RNNs), which is slow and forgetful over long distances. Self-attention instead lets every token directly gather information from every other token in one parallel step — and learn which ones matter.

Query, Key, Value — the lookup metaphor

Attention reframes "understanding context" as a soft dictionary lookup. From each token's embedding, three small learned matrices produce three vectors:

  • Query (Q) — "what am I looking for?" The current token's question.
  • Key (K) — "what do I offer?" A label advertising what each token is about.
  • Value (V) — "what information do I pass on if attended to?" The actual content.

To process a token, you compare its Query against every token's Key with a dot product — high when they align, the way two arrows pointing the same way have a large dot product. Those scores become attention weights (via softmax), and the token's output is the weighted sum of all the Values. A token that matches your query contributes most of its Value; irrelevant tokens contribute almost nothing.

Why the details matter

  • Why the dot product? It measures alignment between a Query and a Key. Learning good Q/K projections is learning "which tokens should talk to which" — grammar, coreference, topic, all emerge here.
  • Why divide by √d? In high dimensions, dot products grow large, which makes softmax razor-sharp (one token gets weight ≈1) and kills the gradient. Scaling by √d keeps the scores tame so attention stays smooth and trainable. Toggle the checkbox to feel the difference.
  • Why softmax? It turns arbitrary scores into a clean probability distribution that sums to 1 — a set of mixing weights for the values.
  • "Self" attention means Q, K, V all come from the same sequence (tokens attend to each other). In cross-attention (e.g. decoder→encoder) the Queries come from one sequence and Keys/Values from another.

Multi-head attention just runs several of these in parallel with different learned projections, so one head can track syntax while another tracks coreference, then their outputs are concatenated. Stack that with a feed-forward network, add residual connections and normalization, and repeat — that's a transformer block.

Takeaways: attention is a soft, learnable lookup. Each token emits a Query, a Key, and a Value; score every Query·Key, scale by √d, softmax into weights, and output the weighted sum of Values.
