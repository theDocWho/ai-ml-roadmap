Support Vector Machines

Of all the lines that separate two classes, which is best? The SVM's answer is elegant: the one with the widest margin — the biggest buffer to the nearest points. And when no straight line works, the kernel trick bends the rules by lifting the data into a higher dimension where a line does.

Widest is best — and only a few points decide it

Many lines separate these two classes, but they're not equally good: a line that skims right past a point will misclassify the next similar example. The SVM picks the boundary that sits as far as possible from both classes — it maximises the margin, the distance to the closest points on each side. Those closest points are the support vectors; remarkably, they alone determine the boundary — move any other point and nothing changes. The kernel trick: make it separable

Real data is rarely so tidy. Consider points on a line where one class sits in the middle and the other on the outside — no single threshold can split them. The trick: add a new feature derived from the old ones (here, x²) and the points lift into a higher dimension where they pull apart, and a straight line separates them again. Project that line back down and it becomes a curved boundary in the original space. The "kernel" is a shortcut that computes distances in that higher space without ever building the coordinates. Takeaways: an SVM chooses the separating boundary with the widest margin; only the closest points — the support vectors — define it. For non-linearly-separable data, the kernel trick maps inputs into a higher-dimensional space where a linear boundary works, giving a curved boundary back in the original space (RBF kernel = "infinite" dimensions). Robust, but slower to train on very large datasets.
