The confusion matrix

"Accuracy" is the metric that lies. The confusion matrix is the honest picture: it splits every prediction into four boxes, and from those four numbers come precision, recall, and F1 — the metrics that actually matter when the classes are imbalanced (fraud, disease, intrusion).

Four outcomes, not one

A binary classifier doesn't just output right/wrong — it can be wrong in two different ways, and they have very different costs. A model usually emits a score; you pick a threshold above which you predict "positive." That choice sorts every example into one of four boxes:

  • True Positive — predicted positive, really positive ✓
  • False Positive — predicted positive, actually negative (a false alarm)
  • False Negative — predicted negative, actually positive (a miss)
  • True Negative — predicted negative, really negative ✓

Precision vs recall — you can't max both

Two ratios read off the matrix capture the trade-off:

  • Precision = TP / (TP + FP) — "when I say positive, how often am I right?" Raising the threshold makes you pickier → precision up, but you miss more.
  • Recall = TP / (TP + FN) — "of all the real positives, how many did I catch?" Lowering the threshold catches more → recall up, but with more false alarms.

Push the threshold to either extreme and you'll see them pull in opposite directions. F1 is their harmonic mean — 2·P·R / (P + R) — a single number that's only high when both are decent, which is why it's the go-to metric for imbalanced problems (exam Q23). Notice how accuracy can stay high even when the model is useless on the rare class: with 95% negatives, "always predict negative" scores 95% accuracy and 0% recall. That's the trap the confusion matrix exposes.

Takeaways: a classifier's score + a threshold produce TP/FP/FN/TN. Precision = TP/(TP+FP) (trust when it says yes), recall = TP/(TP+FN) (coverage of real positives) — raising the threshold trades recall for precision. F1 balances them; prefer it to accuracy when classes are imbalanced. Pick the threshold by which error costs more in your domain.
