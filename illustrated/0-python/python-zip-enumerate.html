zip & enumerate

In Java you loop with an index and reach into arrays: for (int i…) a[i], b[i]. Python considers that a code smell. enumerate gives you the index and the item together; zip walks several sequences in lockstep. Learn these two and you'll almost never write range(len(...)) again.

enumeratezip lockstep iterationunpacking lazy

enumerate: index + item, together

When you genuinely need the position and the value, enumerate hands you both as a pair on each loop, so you never index manually:

for i, name in enumerate(names):  # i = 0,1,2… name = the item
    print(i, name)

enumerate(names, start=1)  # count from 1 instead of 0

zip: walk sequences in lockstep

zip(a, b) yields tuples pairing a[0] with b[0], a[1] with b[1], and so on — perfect for iterating two related lists at once (names & scores, predictions & labels). The one rule to remember: zip stops at the shortest input. Extra items in the longer sequence are silently ignored. Adjust the lengths and watch the pairing — and the leftovers:

Two idioms worth memorising

  • Unpack in the loop header. for i, name in enumerate(...) and for name, score in zip(...) destructure each tuple right where you receive it — clean and readable.
  • zip(*matrix) transposes. Because * unpacks the rows as separate arguments, zip(*rows) pairs up the first elements, then the seconds… turning rows into columns. A genuinely handy one-liner.

Both are lazy (like generators): they produce pairs on demand, so wrap them in list(...) if you want to see or reuse all the results at once.

Takeaways: use enumerate(seq) for index+item (optionally start=) instead of range(len(seq)); use zip(a, b, …) to iterate sequences in lockstep — it stops at the shortest. Unpack the tuples in the loop header; zip(*rows) transposes. Both are lazy — wrap in list() to materialise.
